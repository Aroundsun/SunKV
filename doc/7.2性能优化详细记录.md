## 3. 详细改动说明

## 3.1 网络缓冲区优化

### 3.1.1 `Buffer::retrieve()` 索引修复

文件：`network/Buffer.h`

问题与风险：

- 原实现中 `retrieve(size_t len)` 存在读索引重复递增路径，可能导致读指针越界式前移。

优化内容：

- 调整索引推进逻辑：
  - `len < readableBytes()` 时仅递增一次 `readerIndex_`
  - `len == readableBytes()` 时直接回收到 `kCheapPrepend`

收益：

- 避免索引异常推进
- 保证后续缓冲区回收与读写行为一致

### 3.1.2 `ensureWritableBytes()` 复用前置空间

文件：`network/Buffer.h`

优化内容：

- 旧逻辑在可写不足时倾向于扩容。
- 新逻辑优先判断：
  - `prependableBytes() + writableBytes() >= len + kCheapPrepend`
  - 满足时先 compact（搬移可读数据到前置区后部）而不是直接重新分配。
- 仅在确实不足时再扩容，并保持指数增长策略减少扩容次数。

收益：

- 减少内存重新分配
- 降低复制次数
- 改善高频 append 场景下的吞吐稳定性

## 3.2 锁粒度优化

文件：`storage/ShardedKVStore.cpp`

优化对象：

- `size()`
- `shard_sizes()`
- `clear()`
- `cleanup_expired()`

优化前：

- 部分路径会先持有“所有分片锁”再执行遍历，临界区较大。

优化后：

- 改为逐分片短临界区（按分片循环，加锁 -> 操作 -> 释放）。

收益：

- 降低长时间锁占用
- 减少多线程下阻塞窗口
- 提升统计查询和维护操作对业务路径的友好性

---

## 3.3 系统调用优化

文件：`network/Buffer.cpp`

### 3.3.1 `readFd()` 快速路径

优化内容：

- 当当前可写空间已经足够大时，直接调用 `read()`。
- 仅在空间不足时使用 `readv()`（主缓冲 + 临时缓冲）双段读取。

收益：

- 减少高频路径下 `iovec` 组装开销
- 保持大流量时的兼容读取能力

### 3.3.2 `writeFd()` 统一回收路径

优化内容：

- 写入成功后由 `readerIndex_ += n` 改为调用 `retrieve(n)`。

收益：

- 统一索引管理逻辑
- 避免回收路径分叉导致的状态不一致风险

## 3.4 内存池优化

### 3.4.1 新增独立内存池模块

新增文件：

- `common/MemoryPool.h`
- `common/MemoryPool.cpp`

实现要点：

- 提供 `ThreadLocalBufferPool`（线程本地固定块内存池）
- 提供 `Lease`（RAII 租约）自动归还机制
- 按“块大小”分桶缓存
- 每个桶可配置最大缓存块数量

### 3.4.2 Buffer 接入内存池

文件：`network/Buffer.cpp`

优化内容：

- `readFd()` 的临时 64KB 缓冲由内存池租借，而非函数内临时数组模式。
- 使用 `Lease` 生命周期自动归还。

收益：

- 降低高频路径内存抖动
- 为后续多模块复用（WAL/Snapshot/网络层）提供基础设施

### 3.4.3 内存池统计与容量控制（增强版）

实现能力：

- 统计项：
  - `hit_count`
  - `miss_count`
  - `release_count`
  - `discard_count`
  - `cached_block_count`
- 控制项：
  - `setMaxCachedBlocksPerSize(size_t limit)`
  - `maxCachedBlocksPerSize()`
  - `resetStats()`
  - `getStats()`

作用：

- 支持后续可观测性建设（7.3）
- 支持不同部署环境的内存占用调优

## 6. 兼容性与风险评估

兼容性：

- 本次改动不涉及协议层语义变化。
- 对外接口保持不变。

风险点与应对：

- 风险：内存池策略配置不当导致缓存增长偏大  
  应对：提供桶容量上限与统计项，支持调参与观测。

- 风险：锁粒度调整导致并发行为变化  
  应对：增加并发回归测试，验证功能一致性。

- 风险：系统调用路径分支增加  
  应对：使用 `pipe` 场景测试覆盖 `readFd/writeFd` 核心路径。

本次 7.2 优化已完成四项目标并通过对应测试验证，核心结果如下：

- 缓冲区内存行为更稳定，减少不必要扩容

- 读写系统调用路径更贴合实际场景
- 内存池实现独立化并成功接入 Buffer，为后续性能观测与进一步优化打下基础
