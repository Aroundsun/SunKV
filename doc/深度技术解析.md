## 🔄 EventLoop 深度解析

### 核心设计理念
EventLoop 是 Reactor 模式的核心，每个线程运行一个 EventLoop，实现"one loop per thread"。

### 内部状态管理
```cpp
class EventLoop {
private:
    // 线程标识 - 确保线程安全
    std::atomic<bool> looping_;        // 是否正在循环
    std::atomic<bool> quit_;          // 退出标志
    std::atomic<bool> eventHandling_;  // 是否正在处理事件
    std::atomic<bool> callingPendingTasks_; // 是否正在执行待处理任务
    std::atomic<bool> destructing_;   // 析构标志
    
    // 线程亲和性
    const std::thread::id threadId_;  // 创建线程ID
    
    // 核心组件
    std::unique_ptr<Poller> poller_;           // epoll 封装
    std::unique_ptr<TimerQueue> timerQueue_;   // 定时器队列
    std::unique_ptr<Channel> wakeupChannel_;   // 唤醒通道
    
    // 任务队列 - 跨线程通信
    std::vector<Functor> pendingTasks_;       // 待执行任务
    std::mutex mutex_;                       // 任务队列锁
    
    // 事件通道管理
    std::vector<Channel*> activeChannels_;      // 活跃通道
    Channel* currentActiveChannel_;            // 当前处理通道
};
```

### 关键实现细节

#### 1. 线程安全机制
```cpp
void EventLoop::assertInLoopThread() const {
    if (!isInLoopThread()) {
        // 记录错误信息
        LOG_ERROR("EventLoop::assertInLoopThread() - EventLoop {} was created in thread {} but current thread is {}", 
                 threadId_, std::this_thread::get_id());
        
        // 如果正在析构，不退出程序
        if (!destructing_) {
            exit(1);
        }
    }
}
```

**设计意图**：
- 确保所有操作都在正确的线程中执行
- 防止跨线程访问导致的数据竞争
- 析构时提供优雅降级

#### 2. 跨线程唤醒机制
```cpp
void EventLoop::wakeup() {
    uint64_t one = 1;
    ssize_t n = write(wakeupFd_, &one, sizeof(one));
    if (n != sizeof(one)) {
        LOG_ERROR("EventLoop::wakeup() writes {} bytes instead of 8", n);
    }
}
```

**技术细节**：
- 使用 `eventfd` 创建唤醒文件描述符
- 8字节写入，避免信号量竞争
- 非阻塞写入，确保不会阻塞

#### 3. 事件循环核心逻辑
```cpp
void EventLoop::loop() {
    assert(!looping_);
    assertInLoopThread();
    looping_ = true;
    
    while (!quit_) {
        // 1. 等待事件
        activeChannels_.clear();
        pollReturnTime_ = poller_->poll(kPollTimeMs, &activeChannels_);
        
        // 2. 处理定时器
        for (const auto& channel : activeChannels_) {
            currentActiveChannel_ = channel;
            currentActiveChannel_->handleEvent(pollReturnTime_);
        }
        currentActiveChannel_ = nullptr;
        
        // 3. 执行待处理任务
        doPendingTasks();
    }
    
    looping_ = false;
}
```

**执行顺序的重要性**：
1. 先处理 I/O 事件（高优先级）
2. 再处理定时器（时间敏感）
3. 最后处理用户任务（低优先级）

#### 轮询算法实现
```cpp
EventLoop* EventLoopThreadPool::getNextLoop() {
    baseLoop_->assertInLoopThread();
    assert(started_);
    
    EventLoop* loop = baseLoop_;
    
    if (!loops_.empty()) {
        // 轮询选择下一个线程
        loop = loops_[next_];
        ++next_;
        if (static_cast<size_t>(next_) >= loops_.size()) {
            next_ = 0;  // 循环回到开始
        }
    }
    
    return loop;
}
```

**负载均衡效果**：
- 连接均匀分布到各个 IO 线程
- 避免单个线程过载
- 简单高效，无锁竞争

#### 新连接处理流程
```cpp
void TcpServer::newConnection(int sockfd, const std::string& localAddr, const std::string& peerAddr) {
    loop_->assertInLoopThread();
    
    // 1. 生成连接名称
    char buf[64];
    snprintf(buf, sizeof(buf), "#%d", nextConnId_);
    ++nextConnId_;
    std::string connName = name_ + buf;
    
    // 2. 选择 IO 线程
    EventLoop* ioLoop = threadPool_->getNextLoop();
    
    // 3. 创建连接对象
    TcpConnectionPtr conn = std::make_shared<TcpConnection>(ioLoop, connName, sockfd, localAddr, peerAddr);
    
    // 4. 设置回调函数
    conn->setConnectionCallback(connectionCallback_);
    conn->setMessageCallback(messageCallback_);
    conn->setWriteCompleteCallback(writeCompleteCallback_);
    conn->setCloseCallback(std::bind(&TcpServer::removeConnection, this, std::placeholders::_1));
    
    // 5. 注册连接
    connections_[connName] = conn;
    
    // 6. 在 IO 线程中建立连接
    ioLoop->runInLoop(std::bind(&TcpConnection::connectEstablished, conn));
}
```

**关键设计点**：
1. **连接命名**：确保每个连接有唯一标识
2. **线程选择**：使用轮询算法分发连接
3. **回调设置**：统一设置所有回调函数
4. **线程安全**：在 IO 线程中建立连接

```cpp
void TcpConnection::handleRead(Timestamp receiveTime) {
    loop_->assertInLoopThread();
    int savedErrno = 0;
    ssize_t n = inputBuffer_->readFd(channel_->fd(), &savedErrno);
    
    if (n > 0) {
        // 成功读取数据
        messageCallback_(shared_from_this(), inputBuffer_->peek(), inputBuffer_->readableBytes());
        inputBuffer_->retrieve(n);
    } else if (n == 0) {
        // 连接关闭
        handleClose();
    } else {
        // 读取错误
        errno = savedErrno;
        LOG_ERROR("TcpConnection::handleRead - read error");
        handleError();
    }
}
```

**缓冲区设计优势**：
- 自动扩容，避免数据丢失
- 零拷贝操作，提高性能
- 统一接口，简化使用

#### 写事件处理
```cpp
void TcpConnection::handleWrite() {
    loop_->assertInLoopThread();
    if (channel_->isWriting()) {
        ssize_t n = write(channel_->fd(), outputBuffer_->peek(), outputBuffer_->readableBytes());
        
        if (n > 0) {
            outputBuffer_->retrieve(n);
            if (outputBuffer_->readableBytes() == 0) {
                channel_->disableWriting();
                if (writeCompleteCallback_) {
                    loop_->queueInLoop(std::bind(writeCompleteCallback_, shared_from_this()));
                }
            }
        } else {
            LOG_ERROR("TcpConnection::handleWrite - write error");
        }
    }
}
```

**写缓冲区的作用**：
- 处理写不完整的情况
- 支持批量发送优化
- 避免阻塞事件循环

### 事件类型定义
```cpp
const int kNoneEvent = 0;           // 无事件
const int kReadEvent = EPOLLIN;      // 读事件

```

### 事件处理流程
```cpp
void Channel::handleEvent(Timestamp receiveTime) {
    std::shared_ptr<void> guard;
    if (tied_) {
        guard = tie_.lock();  // 防止对象被析构
        if (guard) {
            handleEventWithGuard(receiveTime);
        }
    } else {
        handleEventWithGuard(receiveTime);
    }
}

void Channel::handleEventWithGuard(Timestamp receiveTime) {
    eventHandling_ = true;
    
    if ((revents_ & EPOLLHUP) && !(revents_ & EPOLLIN)) {
        if (closeCallback_) closeCallback_();
    }
    
    if (revents_ & (EPOLLERR | EPOLLHUP)) {
        if (errorCallback_) errorCallback_();
    }
    
    if (revents_ & EPOLLIN) {
        if (readCallback_) readCallback_(receiveTime);
    }
    
    if (revents_ & EPOLLOUT) {
        if (writeCallback_) writeCallback_();
    }
    
    eventHandling_ = false;
}
```

**事件优先级处理**：
1. **错误事件**：EPOLLERR | EPOLLHUP
2. **读事件**：EPOLLIN
3. **写事件**：EPOLLOUT

### 核心操作实现

#### 更新通道
```cpp
void Poller::updateChannel(Channel* channel) {
    ownerLoop_->assertInLoopThread();
    
    const int index = channel->index();
    
    if (index == kNew || index == kDeleted) {
        // 新通道或已删除通道，添加到 epoll
        int fd = channel->fd();
        if (index == kNew) {
            channels_[fd] = channel;
        }
        
        channel->setIndex(kAdded);
        update(EPOLL_CTL_ADD, channel);
    } else {
        // 已存在的通道，修改 epoll
        update(EPOLL_CTL_MOD, channel);
    }
}
```

#### 等待事件
```cpp
Timestamp Poller::poll(int timeoutMs, std::vector<Channel*>* activeChannels) {
    int numEvents = ::epoll_wait(epollfd_, 
                                events_.get(), 
                                kInitEventListSize, 
                                timeoutMs);
    
    Timestamp now(Timestamp::now());
    
    if (numEvents > 0) {
        for (int i = 0; i < numEvents; ++i) {
            Channel* channel = static_cast<Channel*>(events_[i].data.ptr);
            channel->setRevents(events_[i].events);
            activeChannels->push_back(channel);
        }
    } else if (numEvents == 0) {
        // 超时
    } else {
        // 错误
        if (errno != EINTR) {
            LOG_ERROR("Poller::poll() error");
        }
    }
    
    return now;
}
```

**性能优化点**：
- 使用 `unordered_map` 快速查找 Channel
- 事件数组复用，减少内存分配
- 错误处理完善，避免异常退出

### 定时器管理算法

#### 添加定时器
```cpp
int64_t TimerQueue::addTimer(const TimerCallback& cb, Timestamp when, double interval) {
    std::shared_ptr<Timer> timer = std::make_shared<Timer>(cb, when, interval);
    
    loop_->runInLoop(std::bind(&TimerQueue::addTimerInLoop, this, timer));
    
    return timer->sequence();
}

void TimerQueue::addTimerInLoop(std::shared_ptr<Timer> timer) {
    loop_->assertInLoopThread();
    
    bool earliestChanged = false;
    
    if (timers_.empty() || timer->expiration() < timers_.begin()->first) {
        earliestChanged = true;
    }
    
    timers_.insert(Entry(timer->expiration(), timer));
    
    if (earliestChanged) {
        resetTimerfd(timerfd_, timer->expiration());
    }
}
```

#### 处理过期定时器
```cpp
std::vector<TimerQueue::Entry> TimerQueue::getExpired(Timestamp now) {
    std::vector<Entry> expired;
    Entry sentry(now, std::shared_ptr<Timer>(reinterpret_cast<Timer*>(UINTPTR_MAX)));
    
    auto end = timers_.lower_bound(sentry);
    std::copy(timers_.begin(), end, std::back_inserter(expired));
    timers_.erase(timers_.begin(), end);
    
    return expired;
}
```

**算法复杂度**：
- 插入：O(log n)
- 删除：O(log n)
- 查找最早定时器：O(1)

```
│  │ prepend  │    readable     │    writable        │  │
```

### 关键操作实现

#### 数据写入
```cpp
void Buffer::append(const char* data, size_t len) {
    ensureWritableBytes(len);
    std::copy(data, data + len, begin() + writerIndex_);
    writerIndex_ += len;
}

void Buffer::ensureWritableBytes(size_t len) {
    if (writableBytes() < len) {
        size_t readable = readableBytes();
        
        // 移动数据到开头
        std::copy(begin() + readerIndex_, begin() + writerIndex_, begin());
        
        readerIndex_ = 0;
        writerIndex_ = readable;
        
        // 扩容
        buffer_.resize(writerIndex_ + len);
    }
}
```

#### 数据读取
```cpp
ssize_t Buffer::readFd(int fd, int* savedErrno) {
    char extrabuf[65536];  // 栈上临时缓冲区
    struct iovec vec[2];
    
    size_t writable = writableBytes();
    vec[0].iov_base = begin() + writerIndex_;
    vec[0].iov_len = writable;
    vec[1].iov_base = extrabuf;
    vec[1].iov_len = sizeof(extrabuf);
    
    const int iovcnt = (writable < sizeof(extrabuf)) ? 2 : 1;
    ssize_t n = readv(fd, vec, iovcnt);
    
    if (n < 0) {
        *savedErrno = errno;
    } else if (static_cast<size_t>(n) <= writable) {
        writerIndex_ += n;
    } else {
        writerIndex_ = buffer_.size();
        append(extrabuf, n - writable);
    }
    
    return n;
}
```

**性能优化**：
- 使用 `readv` 减少系统调用
- 栈上临时缓冲区避免内存分配
- 自动扩容机制平衡内存使用

```cpp
class EventLoopThread {
private:
    EventLoop* loop_;                    // 事件循环指针
    std::thread thread_;                // 线程对象
    std::mutex mutex_;                  // 互斥锁
    std::condition_variable cond_;        // 条件变量

EventLoop* EventLoopThread::startLoop() {
    thread_ = std::thread(&EventLoopThread::threadFunc, this);
    
    EventLoop* loop = nullptr;
    {
        std::unique_lock<std::mutex> lock(mutex_);
        while (loop_ == nullptr) {
            cond_.wait(lock);  // 等待线程初始化完成
        }
        loop = loop_;
    }
    
    return loop;
}

void EventLoopThread::threadFunc() {
    EventLoop loop;
    
    if (callback_) {
        callback_(&loop);  // 执行初始化回调
    }
    
    {
        std::lock_guard<std::mutex> lock(mutex_);
        loop_ = &loop;
        cond_.notify_one();  // 通知主线程
    }
    
    loop.loop();  // 运行事件循环
    
    std::lock_guard<std::mutex> lock(mutex_);
    loop_ = nullptr;
}
```

**线程同步机制**：
- 使用条件变量等待线程初始化完成
- 互斥锁保护共享数据
- 确保线程安全的事件循环返回
