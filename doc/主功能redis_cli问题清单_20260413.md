# 主功能 redis-cli 问题清单（2026-04-13）

本文档用于记录 `scripts/main_functional_redis_cli.sh` 的实测失败项与关联高风险问题，作为后续逐步修复的执行清单。

---

## 一、实测失败项（脚本结果）

基于脚本 `scripts/main_functional_redis_cli.sh` 的一次完整运行，结果：`FAILED(7)`。

### 1) 列表命令行为异常
- **复现命令**：
  - `LPUSH l1 c b a`（期望返回 `3`，实际 `4`）
  - `LLEN l1`（期望 `3`，实际 `4`）
  - `RPOP l1`（期望 `c`，实际空）
  - 后续 `LLEN l1`（期望 `1`，实际 `2`）
- **初步判断**：
  - LIST 在“键已是 STRING 时转 LIST”的逻辑中，把旧字符串值也加入了列表，导致长度与元素序列偏移。
- **优先级**：P0

### 2) 集合命令计数异常
- **复现命令**：
  - `SADD s1 x y x`（通过）
  - `SCARD s1`（期望 `2`，实际 `3`）
- **初步判断**：
  - SET 在“键已是 STRING 时转 SET”的逻辑中，把旧字符串值插入集合，导致基数偏大。
- **优先级**：P0

### 3) 哈希命令计数异常
- **复现命令**：
  - `HSET h1 f1 v1`
  - `HSET h1 f2 v2`
  - `HLEN h1`（期望 `2`，实际 `3`）
- **初步判断**：
  - HASH 在“键已是 STRING 时转 HASH”的逻辑中，注入了保底字段（例如 `value`），导致字段数偏大。
- **优先级**：P0

### 4) `DBSIZE` 与预期不一致
- **复现命令**：
  - 在上述操作后 `DBSIZE`（期望 `4`，实际 `3`）
- **初步判断**：
  - 与前面 LIST/SET/HASH 的数据形态偏移叠加有关，也可能存在过期键清理时序影响。
- **优先级**：P1

## 二、已识别的高风险结构问题（待修）

### 5) 写命令持久化覆盖不完整
- **位置**：`server/Server.cpp`（`processCommand()`）
- **现象**：
  - `SET/DEL` 会写 WAL；
  - 多数集合类/哈希类/TTL 变更命令没有对应 WAL 落盘。
- **风险**：重启后状态不一致、数据丢失。
- **优先级**：P0

### 6) TCP 启动路径重复
- **位置**：`server/Server.cpp`
- **现象**：
  - `initializeNetwork()` 与 `setupConnectionCallbacks()` 都调用了 `tcp_server_->start()`。
- **风险**：启动时序不一致，潜在竞态。
- **优先级**：P1

### 7) 线程池控制双轨
- **位置**：
  - `server/Server.cpp`（Server 外部线程池）
  - `network/TcpServer.cpp`（TcpServer 内部线程池）
- **风险**：配置语义与实际执行可能不一致。
- **优先级**：P1

## 三、建议修复顺序（执行计划）

1. **先修功能正确性（P0）**
   - LIST/SET/HASH 的“STRING -> 复合类型”转换策略改为不注入旧字符串值；
   - 修正对应返回值与计数行为；
   - 复跑 `main_functional_redis_cli.sh`，目标失败项先降到 0（至少功能层）。

2. **补齐持久化一致性（P0）**
   - 所有写命令统一接入 WAL 落盘；
   - 增加“重启后一致性”断言。

3. **收敛结构双轨（P1）**
   - 收敛 `tcp_server_->start()` 单入口；
   - 统一线程池来源与配置。

---

## 四、验收标准

- `scripts/main_functional_redis_cli.sh` 返回 `OK`；
- 核心命令行为与 Redis 预期一致（返回值、计数、元素顺序）；
- 重启后关键数据与命令结果一致；
- 不再出现“写了内存但没落 WAL”的主路径。
