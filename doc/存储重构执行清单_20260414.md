# SunKV 存储重构执行清单（2026-04-14）

## 目标

- 不是“删路径收敛”，而是完成一次可演进的存储架构重构。
- 保留 `StorageEngine` 作为统一存储门面，逐步吸收当前 `multi_storage_` 能力。
- 最终实现：Server 不直连底层容器；持久化边界清晰；命令语义单一。
