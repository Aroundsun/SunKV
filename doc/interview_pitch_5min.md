# SunKV 5 分钟面试讲解提纲

## 0:00 - 0:40 项目定位

- SunKV 是一个 C++17 的 RESP 兼容 KV 系统，面向单机高并发场景。
- 我重点做的是“功能扩展 + 稳定性回归 + 工程化交付”闭环，不只是写命令。

## 0:40 - 2:10 技术亮点

- 协议与连接：增量解析 RESP，处理半包/粘包、异常帧和连接生命周期。
- 事务与状态机：实现 `MULTI/EXEC/DISCARD`，包含队列执行与错误路径。
- Pub/Sub：实现订阅态门禁、广播投递、断连自动清理。
- 可观测性：统一命令埋点 + 慢日志 + `STATS key=value` 指标输出。

## 2:10 - 3:30 工程与质量

- 把测试体系按 `client/network/protocol/server/storage2` 分层。
- 补了异常回归：非法 RESP、输入缓冲溢出、半关闭与中断恢复。
- CI 从单链路扩展为 fast/full，PR 快反馈，主干全量兜底，并保留失败日志。

## 3:30 - 4:30 演示方式

- 一条命令跑通：`./scripts/demo_all_in_one.sh`
- 演示流程：启动 -> 功能回归 -> 压测 -> 停止 -> 自动生成摘要。
- 面试现场可直接展示 runbook 与日志路径，证明结果可复现。

## 4:30 - 5:00 收尾

- 我在这个项目里的重点是：定义边界、控制回归风险、把实现结果转成可验证证据。
- 下一步会继续做性能趋势化和 client 重连稳定性增强。
