# SunKV 性能优化（全量迁移去重版）

---

## 1. 目标与评估口径

## 1.1 阶段目标

- QPS >= 50,000
- 延迟 < 1ms
- 并发连接 >= 1000

## 1.2 压测口径

- `redis-benchmark -n 50000 -c 50 -t set,get -q`
- `redis-benchmark -n 50000 -c 50 -t set,get --csv`
- Redis vs SunKV 对比口径：`-n 200000 -c 50 -t set,get`，模式 `P=1/P=16`


---

## 2. 结论

1. 从约 `3k rps` 提升到常态 `24k~33k rps` 区间，显著提升但未达 `50k`。

4. RESPParser 连接级复用在当前环境下“工程收益明确、吞吐近似持平”，主瓶颈不在 parser 构造。
5. 当前主瓶颈集中在高频链路：`onMessage -> processCommand -> sendInLoop -> write`。

## 4.详细优化

### 4.1 网络缓冲区优化

文件：`network/Buffer.h`

- `Buffer::retrieve()` 索引修复：
  - `len < readableBytes()` 仅递增一次 `readerIndex_`
  - `len == readableBytes()` 直接回收到 `kCheapPrepend`
- `ensureWritableBytes()`：
  - 满足 `prependableBytes() + writableBytes() >= len + kCheapPrepend` 时先 compact
  - 仅确实不足时扩容，并保留指数增长策略

收益：

- 避免索引异常推进
- 减少内存重分配与复制
- 改善高频 append 场景稳定性

### 4.2 锁粒度优化

文件：`storage/ShardedKVStore.cpp`

优化对象：

- `size()`
- `shard_sizes()`
- `clear()`
- `cleanup_expired()`

策略：由全分片长临界区改为逐分片短临界区。

收益：

- 降低长锁占用
- 减少并发阻塞窗口

### 4.3 系统调用优化

文件：`network/Buffer.cpp`

- `readFd()`：
  - 可写空间充足走 `read()`
  - 不足时走 `readv()`（主缓冲 + 临时缓冲）
- `writeFd()`：
  - 写成功后统一 `retrieve(n)` 回收

收益：

- 降低 `iovec` 组装开销
- 索引管理路径一致

### 4.4内存池优化

新增：

- `common/MemoryPool.h`
- `common/MemoryPool.cpp`

实现要点：

- `ThreadLocalBufferPool` 线程本地固定块池
- `Lease` RAII 自动归还
- 按块大小分桶
- 每桶缓存块上限可配置

Buffer 接入：

- `network/Buffer.cpp` 的 `readFd()` 临时 64KB 缓冲改为租借池内存

## 6. RESPParser 连接复用

背景：

- `Server::onMessage` 原先每条命令临时构造 parser,耗费资源，不利于后续扩展
- 目标改为每连接一个状态机复用，也方便后续扩展

改动：

- `ConnParseState` 新增 `RESPParser parser` 与 `pending_input`
- `conn_inbuf_`：`unordered_map<string,string>` -> `unordered_map<string,shared_ptr<ConnParseState>>`
- 使用 `processed_bytes` 差值推进消费，单命令完成后 `parser.reset()`
- 连接关闭时释放该连接解析状态

涉及文件：

- `server/Server.h`
- `server/Server.cpp`
