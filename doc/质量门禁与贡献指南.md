# 质量门禁与贡献指南（SunKV）

本页说明如何在本地运行 **format / lint / test**，以及 CI 会做哪些检查。

## 1. 快速开始（本地）

### 构建 + 测试

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DBUILD_TEST=ON -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
cmake --build build -j"$(nproc)"
ctest --test-dir build --output-on-failure
```

### 仅跑某类测试（按 label）

```bash
ctest --test-dir build -L network --output-on-failure
ctest --test-dir build -L storage2 --output-on-failure
```

## 2. clang-format（格式化）

仓库根目录提供 `.clang-format`。

### 本地格式化单个文件

```bash
clang-format -i path/to/file.cpp
```

### CI 的格式检查

CI 会运行 `scripts/clang_format_check.sh`：默认只检查 **相对基线分支（如 origin/main）有改动的 C/C++ 文件**，避免一次性重排历史代码。

## 3. clang-tidy（静态检查）

仓库根目录提供 `.clang-tidy`。

### 本地运行（建议仅对改动文件）

需要 `compile_commands.json`（配置时加 `-DCMAKE_EXPORT_COMPILE_COMMANDS=ON`）。

```bash
./scripts/clang_tidy_check.sh
```

该脚本默认只检查 `network/`、`server/`、`storage2/`、`common/`、`protocol/` 下的改动文件。

## 4. CI 做了什么

GitHub Actions workflow（`.github/workflows/cmake-single-platform.yml`）会执行：

- 安装依赖（含 `clang-format`、`clang-tidy`、`libspdlog-dev`、`ripgrep`）
- CMake configure（`BUILD_TEST=ON`，并导出 `compile_commands.json`）
- `clang-format` 检查（改动文件）
- `clang-tidy` 检查（改动文件，核心目录）
- build
- `ctest --output-on-failure`

## 5. 约定与建议

- **不要引入 `exit(1)`** 这种库层强退；倾向抛异常或向上层返回错误，由 `main`/`Server` 统一处理。
- 新增/修改功能请同步：
  - 最小测试（单元/集成）
  - 关键行为文档（如配置项、兼容性差异）
