# SunKV 测试文件使用说明

本文档用于记录 `test/` 目录下每个测试文件的用途与使用方式，便于日常开发、回归验证和 CI 排障。

---

## 1. 通用使用方式

## 1.1 构建

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build -j$(nproc)
```

## 1.2 运行全部测试

```bash
ctest --test-dir build --output-on-failure
```

## 1.3 按模块运行

```bash
ctest --test-dir build -L client --output-on-failure
ctest --test-dir build -L network --output-on-failure
ctest --test-dir build -L protocol --output-on-failure
ctest --test-dir build -L server --output-on-failure
ctest --test-dir build -L storage2 --output-on-failure
```

## 1.4 单测定位方式

- 通过测试名运行：`ctest --test-dir build -R <test_name> --output-on-failure`
- 通过可执行运行：`./build/<target_name>`
- 大多数 `<target_name>` 与文件名一致（去掉 `.cpp`）

## 8. 常见问题与排障

- 端口占用：清理残留进程后重跑
- 测试偶发超时：优先使用单测重跑并加 `--output-on-failure`
- 某一模块批量失败：先跑该模块最小 smoke，再缩小到单文件
- 与本地环境相关的波动（性能/时间敏感）：优先看功能断言，不以单次耗时绝对值判定
