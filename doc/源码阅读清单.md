# SunKV 源码阅读清单（全面版）

## 1. 阅读目标

本清单用于帮助你系统理解 SunKV 的完整实现，重点覆盖：

- 网络事件驱动模型（Reactor + epoll）
- RESP 协议解析与命令执行主链路
- 存储与持久化机制（WAL/Snapshot/Recovery）
- 并发安全、性能优化与测试方法

建议按照本文顺序阅读，不要跳步。

---

## 2. 总体阅读策略

- 先看主流程，再看细节实现，最后看优化与测试。
- 每读完一个模块，必须回答“本模块输入是什么、输出是什么、状态如何管理”。
- 每个阶段都做 1 页笔记，记录：
  - 关键类
  - 关键函数
  - 线程归属
  - 常见错误点

---

## 3. 推荐顺序与时间安排

- 第 1 天：启动流程 + 网络骨架
- 第 2 天：连接收发 + 协议解析 + 命令处理
- 第 3 天：存储引擎 + TTL + 多类型数据
- 第 4 天：持久化与恢复
- 第 5 天：性能优化点 + 测试体系 + 总结复盘
